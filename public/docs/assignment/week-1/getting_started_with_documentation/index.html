    Getting Started with Documentation#

    After creating a basic HTML website to understand how webpages work, I explored better ways to manage and scale documentation for Fab Academy.

    During this process, I had a conversation with my dear friend Rishi Krishna
    (https://rishikrishna.com/), who introduced me to Hugo, a simple and powerful static site generator.

    This marked the transition from a basic HTML-only approach to a more structured and maintainable documentation workflow.

    Why Use Hugo for Documentation?#

    Hugo is a static site generator that converts Markdown files into a complete website.

    Advantages of using Hugo:

    • Content is written in simple Markdown
    • Automatically generates navigation and pages
    • Easy to scale for weekly documentation
    • Separates content from design
    • Fast build and load times

    This makes Hugo well-suited for Fab Academy documentation, which involves large amounts of structured content.

    Installing Hugo on Windows#

    Step 1: Download Hugo#

    1. Open a browser
    2. Go to the official Hugo website:
      https://gohugo.io/
    3. Download the Windows (extended) version
    4. Extract the downloaded ZIP file

    Step 2: Add Hugo to System PATH#

    1. Move the extracted hugo.exe file to a folder (for example: C:\Hugo\bin)
    2. Open System Properties → Environment Variables
    3. Add the folder path to the Path variable
    4. Restart VS Code or the terminal

    To verify installation, open Git Bash or Command Prompt and run:

    hugo version

    Creating a New Hugo Site#

    Open Visual Studio Code and open the folder where you want to create your site.

    Run the following command in the terminal:

    hugo new site nadec-biju

    This creates a new Hugo site with the following structure:

    my-fab-docs/
├── archetypes/
├── content/
├── layouts/
├── static/
├── themes/
└── hugo.toml

    Understanding the Hugo Folder Structure#

    Folder/File Purpose
    content/ All documentation pages go here (written in Markdown)
    static/ Images, files, and downloadable assets
    themes/ Website appearance and layout
    hugo.toml Main configuration file for the site

    Using Hugo with VS Code#

    1. Open the Hugo site folder in Visual Studio Code
    2. Install useful extensions:
      • Markdown Preview
      • Hugo Syntax Highlighting (optional)

    To start the local development server:

    hugo server

    This allows live preview of changes in the browser.

    Writing Content Using Markdown#

    Hugo uses Markdown to create pages.

    To create a new page:

    hugo new week-01/index.md

    This creates a Markdown file inside the content/ folder.

    Basic Markdown Syntax#

    Headings#

    # Heading 1
## Heading 2
### Heading 3

    Paragraph#

    This is a paragraph of text.

    Lists#

    - Item 1
- Item 2
- Item 3

    Links#

    [Fab Academy](https://fabacademy.org)

    Images#

    ![Alt text](/images/sample.png)

    Using My Fab Academy Website as Reference#

    My Fab Academy documentation site follows this structure:

    • Each week has a dedicated page
    • Subpages are used for detailed topics
    • Images are stored inside the static/ folder
    • Markdown is used consistently for clarity

    This structure helps keep documentation readable, navigable, and scalable as the course progresses.

    Where to Add What (Quick Guide)#

    Content Type Location
    Weekly content content/week-XX/
    Images static/images/
    Downloads / files static/files/
    Navigation & layout themes/

    Pushing Only the public Folder to Git#

    When using Hugo, the website is generated as static files inside the public/ folder.
    These files contain the final HTML, CSS, and assets that are actually served on the web.

    The remaining Hugo files (content, themes, layouts, etc.) are source files used to generate the site and are not required for deployment.

    For Fab Academy documentation, only the generated output needs to be pushed.

    Why Only the public Folder Is Pushed#

    • Fab Academy servers expect static HTML files
    • Hugo regenerates the public folder every time
    • Keeps the repository clean and lightweight
    • Avoids unnecessary theme and build files in version control

    Creating a .gitignore File#

    To prevent Git from tracking unwanted files and folders, a .gitignore file is used.

    Step 1: Create the .gitignore File#

    1. Open the Hugo project folder in VS Code
    2. Create a new file named:
    .gitignore

    Step 2: Add Ignore Rules#

    Add the following content to the .gitignore file:

    # Ignore Hugo source folders
content/
themes/
layouts/
archetypes/
resources/

# Ignore configuration and cache files
.hugo_build.lock
node_modules/

# Ignore everything by default
*

# But allow the public folder
!public/
!public/**

    This configuration tells Git to:

    • Ignore all files and folders
    • Track only the public folder and its contents

    Generating the public Folder#

    Before pushing changes, generate the site:

    hugo

    This updates the public folder with the latest version of the site.

    Pushing the public Folder to Git#

    After generating the site:

    git status
git add public
git commit -m "Update generated site"
git push

    Only the public folder will be committed and pushed.

    Important Note#

    The public folder is automatically regenerated by Hugo.
    Any manual edits inside public may be overwritten the next time hugo is run.

    All content changes should always be made inside the Markdown files.

    Hugo Workflow for Setting Up a New Page#

    Creating new pages in Hugo follows a consistent workflow. Here’s how to add new content to your site:

    Step 1: Create a New Page#

    Use the Hugo CLI to create a new page:

    hugo new docs/assignment/week-2/_index.md

    This creates a new markdown file with default front matter.

    Step 2: Edit the Front Matter#

    Every Hugo page starts with front matter (metadata) at the top:

    +++
title = "Week 2 - Computer-Aided Design"
weight = 2
draft = false
+++
    Parameter Purpose
    title Page title shown in sidebar and page header
    weight Controls order in sidebar (lower = higher position)
    draft Set to false to publish, true to hide

    Step 3: Add Markdown Content#

    Below the front matter, write your content using Markdown:

    # Week 2 - Computer-Aided Design

## Introduction

This week I learned about CAD tools...

## Tools Used

- Fusion 360
- SolidWorks

    Step 4: Add Images#

    1. Place images in static/images/ folder
    2. Reference them in markdown:
    ![CAD Screenshot](/images/week2-cad.png)

    Step 5: Preview Changes#

    Run the Hugo server to see changes live:

    hugo server

    Open http://localhost:1313 in your browser.

    Step 6: Generate and Push#

    When satisfied with changes:

    hugo
git add public
git commit -m "Add Week 2 documentation"
git push

    Where to Use Markdown vs HTML#

    Hugo supports both Markdown and raw HTML in content files.

    Use Case Format Example
    Regular text, headings, lists Markdown ## Heading, - list item
    Images Markdown ![alt](/images/pic.png)
    Links Markdown [text](url)
    Tables Markdown `
    Custom layouts, animations HTML + CSS <div class="custom">
    Interactive elements HTML + JavaScript <script>...</script>

    Rule of thumb: Use Markdown for content, HTML for custom visual elements.

    Carousel JavaScript#

    <script>
let currentSlide = 0;
let autoSlideInterval;

function showSlide(n) {
  const slides = document.querySelectorAll('.carousel-slide');
  if (n >= slides.length) { currentSlide = 0; }
  if (n < 0) { currentSlide = slides.length - 1; }
  
  slides.forEach(slide => slide.classList.remove('active'));
  if (slides.length > 0) {
    slides[currentSlide].classList.add('active');
  }
}

function changeSlide(n) {
  currentSlide += n;
  showSlide(currentSlide);
  resetAutoSlide();
}

function autoSlide() {
  currentSlide++;
  showSlide(currentSlide);
}

function resetAutoSlide() {
  clearInterval(autoSlideInterval);
  autoSlideInterval = setInterval(autoSlide, 4000);
}

document.addEventListener('DOMContentLoaded', function() {
  showSlide(currentSlide);
  autoSlideInterval = setInterval(autoSlide, 4000);
});
</script>

    Timeline JavaScript (Scroll Animation)#

    <script>
(function() {
  var items = document.querySelectorAll('.timeline-item');
  var observer = new IntersectionObserver(function(entries) {
    entries.forEach(function(entry) {
      if (entry.isIntersecting) {
        entry.target.classList.add('visible');
      }
    });
  }, { threshold: 0.2, rootMargin: '0px 0px -50px 0px' });
  
  items.forEach(function(item) {
    observer.observe(item);
  });
})();
</script>

    ChatGPT Prompts Used#

    Here are the prompts I used to generate the carousel and timeline components:

    Prompt for Image Carousel#

    Create a responsive image carousel/slider for a Hugo markdown page with:
- Fade transition between slides
- Previous/Next navigation arrows
- Auto-slide every 4 seconds
- Mobile responsive design
- Dark mode support
- Clean modern styling with rounded corners and shadow
- Pure HTML, CSS, and JavaScript (no external libraries)

    Prompt for Timeline Component#

    Create a vertical timeline component for a Hugo markdown page to display 
work experience with:
- Animated appearance on scroll using Intersection Observer
- Gradient line connecting timeline items
- Pulsing dot animation for each entry
- Date badge, title, and description for each item
- Hover effects on timeline cards
- Dark mode support
- Mobile responsive layout
- Pure HTML, CSS, and JavaScript (no external libraries)

    Tips for Using ChatGPT for Web Components#

    1. Be specific about the styling you want (colors, shadows, animations)
    2. Mention “no external libraries” if you want pure HTML/CSS/JS
    3. Request dark mode support for Hugo themes that support it
    4. Ask for mobile responsiveness explicitly
    5. Iterate - start with basic version, then add features
